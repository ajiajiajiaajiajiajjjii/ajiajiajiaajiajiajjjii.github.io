Descrizione sintetica in italiano I sistemi di raccomandazione per l’ingegneria del software (RSSE) svolgono un ruolo
fondamentale nel supportare lo sviluppo del software, offrendo assistenza come il
suggerimento di API o snippet di codice sorgente adattati a un determinato contesto.
La ricchezza di dati disponibili da piattaforme come GitHub e Stack Overflow,
insieme ai progressi nelle tecniche di apprendimento automatico, ha favorito progressi
significativi in questo settore. L'obiettivo principale del progetto è migliorare
l'affidabilità degli RSSE. Nello specifico, i contributi del progetto si concentreranno
sullo sviluppo di strumenti software capaci di creare set di dati per addestrare RSSE
ponendo particolare attenzione ad aspetti di vulnerabilità e privacy. Inoltre, il progetto
svilupperà un “modello di affidabilità” di RSSE e avvierà l’implementazione di RSSE
che sfruttino questo modello
Descrizione sintetica in inglese Recommender Systems for Software Engineers (RSSEs) play a vital role in supporting
software development, offering assistance such as suggesting APIs or source code
snippets tailored to a given context. They also contribute to change management tasks
by efficiently assigning changes to suitable experts. The wealth of data available from
platforms like GitHub and Question & Answer (Q&A) forums such as Stack
Overflow, coupled with advancements in machine learning techniques, has propelled
significant progress in this domain. The primary objective of the project is to enhance
the trustworthiness of RSSEs. Specifically, the contributions of project will focus on
the development of vulnerability and privacy-aware miners capable of creating
training sets for RSSEs. Additionally, the project aims to formulate a comprehensive
trustworthiness model for RSSEs and initiate the implementation of trustworthy
RSSEs that leverage this trustworthiness model
